  • What is happening in this code?

      >>> width = 2.5
      >>> height = 4
      >>> area = width * height
      >>> print(area)

    Answer:  The value 2.5 is being stored in a variable called width on line 1
    The value 4 is being stored in a variable called height in line 2
    The product of the values contained in width * height is being stored in a variable called area
    The value of the variable area is printed to the python interpreter output console in line 4

  • What is the difference between these three statements?

      >>> Llamas in Pyjamas
      >>> "Llamas in Pyjamas"
      >>> # Llamas in Pyjamas

    Answer:  The command in line 1 generates an error because none of the tokens / words in the command are actual python commands
    The command in line 2 is a string literal because its a string surrounded by double quotes, but it doesn't really do anything but echo out to the console the contents between the two double quotes.
    The command in line 3 is a comment because of the pound sign # at the beginning of the line resulting an an ellipsis in the output and if you hit enter again you get right back to the python prompt

  • What is the difference between the two print calls?

      >>> area = 50
      >>> print(area)
      >>> print("area")
    Python 2 vs Python 3: In Python 2, print is a statement rather than a function. So rather than print(area) with parentheses, you can write print area.

    Answer:  The first print call outputs the value stored in the variable area to the console and the second print call outputs the string 'area' t the console.

  • What happens when you run this statement, and why?

      >>> "area" = 50
    Answer: A syntax error occurs after executing this command because when executed it tries to stuff the integer value 50 into a string literal "area" which is not a variable but a literal string value.

  • What would you expect these statements to do? What is the difference between them?

      >>> "Na" * 100 + "Batman"
      >>> "Na" * 100.0 + "Batman"

    Answer:  I was not sure what to expect.  Turns out that the first command above concatenates the string Na to itself 100 times and then concatenates the string Batman to the end and echoes the concatenated string to the console.
    The second command results in a syntax error because the iteration value is of type float which cannot be used as a data type as a string concatenation iterator.

    • What do the numbers in the square brackets do?

        >>> word = "floccinaucinihilipilification"
        >>> word[10:3:-1]

      Answer:  
      The first line stuffs a string in to a variable named word.
      The second line uses the python list slice notation to configure 1.) how many list items from the start of the list to count beginning with the 0th list item, 2.) How many list items to leave off at the end of the list, again using the 0th item as the first number in the count instead of starting at 1, in the 3.) direction and amount of list items to include in each step (value) where a positive value for step is left to right and negative is right to left as such:
      floccinaucinihilipilification




      		<---- :-1] means order right to left <---- 

















      resulting in the following output re-ordered left to right of the right to left order of the captured value:
      icuanic

      per the following Stack Overflow explanation on Python List Slicing Syntax:
      http://stackoverflow.com/questions/509211/explain-pythons-slice-notation
      a[start:end] # items start through end-1
a[start:]    # items start through the rest of the array
a[:end]      # items from the beginning through end-1
a[:]         # a copy of the whole array

      There is also the step value, which can be used with any of the above:

      a[start:end:step] # start through not past end, by step

      The key point to remember is that the :end value represents the first value that is not in the selected slice. So, the difference beween end and start is the number of elements selected (if step is 1, the default).

      The other feature is that start or end may be a negative number, which means it counts from the end of the array instead of the beginning. So:

      a[-1]    # last item in the array
a[-2:]   # last two items in the array
a[:-2]   # everything except the last two items
      Python is kind to the programmer if there are fewer items than you ask for. For example, if you ask for a[:-2] and a only contains one element, you get an empty list instead of an error. Sometimes you would prefer the error, so you have to be aware that this may happen.
